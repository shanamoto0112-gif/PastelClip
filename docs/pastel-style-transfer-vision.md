# パステル画風スタイル変換 — 方向性メモ

**更新**: ユーザー確認済み（実写の同一性を保つ）

---

## やりたいこと

| ❌ やらないこと | ✅ やること |
|---|---|
| 雰囲気フィルター（ぼかし・色味だけ） | **実写をパステル画風に描き直す見た目** |
| AI で似た絵を新規生成（顔・ポーズが変わる） | **同じ顔・体の向き・風景を維持** |
| 静止画だけのイラスト化 | **動画として動き続ける** |

## なぜ「実写 → パステル → 実写」なのか

中間だけ別の絵を生成すると、**別人・別背景**になってしまう。  
**同じ映像**の見た目だけがパステル画に変わり、また実写に戻る — ここに PastelClip の意味がある。

```
[0〜N秒]   オリジナル実写（そのまま）
[N〜中間]  同じフレームをパステル画風にスタイル変換
[末尾M秒] オリジナル実写に復帰
```

## ターゲットの見た目（イメージ B 寄り）

- パステル・色鉛筆の**ストローク感**
- **黒い輪郭線なし**、色の重なりで形が出る
- **淡い**パステル色（ピンク・白・薄緑・空色）
- 参考: `docs/reference/` 内の元画像 + 出力プレビュー

## 技術の方向性（段階的）

### Phase 2a — 静止画プレビュー（今）

- `tools/preview_pastel_style.py`
- OpenCV による NPR（非フォトリアル描画）
- 構図は100%維持、見た目だけ変更

### Phase 2b — 短い動画

- フレームごとに同じ変換を適用
- **時間方向のちらつき**（ストロークが毎フレーム変わる）を抑える

### Phase 2c — PastelClip 本番

- ブラウザ内で回るか検証（重ければ WebGL / ONNX / ハイブリッド）
- 強度プリセット: 大げさ 〜 格調高い

### 避ける方向（当面）

- テキスト/画像からの生成 AI（構図が一致しない）
- `eq` + `gblur` のみのフィルター（現 Phase 1）
