## 1. アプリ概要

### 1.1 アプリ名（仮）
**PastelClip**（仮称・後で変更可）

### 1.2 コンセプト
撮影済みのショート動画をアップロードするだけで、
「**冒頭は実写 → 中間はパステル調 → 末尾は実写**」という
映像表現を自動付与してダウンロードできるWebツール。

TikTok・Instagram Reels・YouTube Shortsへの投稿を想定した
縦型ショート動画向けに最適化する。

### 1.3 ターゲットユーザー
- **メインターゲット**: 中高生（スマホユーザー）
- **動作確認用**: 開発者自身

### 1.4 ターゲットを踏まえた設計方針
- URLを開くだけで使える（インストール不要）
- 操作は30秒以内に完結する
- スマホファースト（縦型UI）
- 動画がサーバーに送られない（プライバシー配慮）

## 2. 処理方式

### ブラウザ版（FFmpeg.wasm）を採用

| 項目 | 内容 |
|---|---|
| **処理場所** | ユーザーのブラウザ内（サーバー不要） |
| **技術** | FFmpeg.wasm（WebAssemblyでFFmpegをブラウザ実行） |
| **理由** | 中高生がURLを開くだけで使える・動画がサーバーに届かない |

### フェーズごとの方針

| フェーズ | 処理方式 | 理由 |
|---|---|---|
| **Phase 1** | ブラウザ版（FFmpeg.wasm） | PCで動作確認 |
| **Phase 2** | スマホ実機テスト | 重さ・速度を検証 |
| **Phase 3** | 重ければサーバーハイブリッドへ切替 | 必要に応じて |

## 4. エフェクト仕様

### 4.1 エフェクトの流れ

```
[0秒] ──── 実写（オリジナル）
      ↓ フェードイン（トランジション 0.5秒）
[N秒] ──── パステル調（フィルター適用中）
      ↓ フェードアウト（トランジション 0.5秒）
[末尾M秒前] ── 実写（オリジナルに戻る）
[終了]
```

- **N**（冒頭実写の長さ）: デフォルト2秒、ユーザーが1〜5秒で指定可
- **M**（末尾実写の長さ）: デフォルト2秒、ユーザーが1〜5秒で指定可
- **トランジション**: フェード（0.5秒）でなめらかに切り替わる

### 4.2 パステル調フィルターの内容

**Instagramのパステルフィルターに近い仕上がり**を目指す。
アニメ・イラスト風にはしない。実写の雰囲気を保ちつつ「やわらかく」する。

| パラメータ | 値（デフォルト） | 説明 |
|---|---|---|
| 彩度（Saturation） | 0.6（元の60%） | 鮮やかさを落とす |
| 明度（Brightness） | 1.2（元の120%） | 全体を明るくする |
| コントラスト（Contrast） | 0.9（元の90%） | わずかにやわらかく |
| ガンマ補正（Gamma） | 1.1 | ハイライトをふわっとさせる |

> ※ Phase 2以降でユーザーがスライダーで強度調整できるようにする

### 4.3 FFmpegフィルター実装方針

- **ツール**: FFmpeg.wasm（ブラウザ内実行）
- **フィルター**: `hue`・`eq`フィルターを組み合わせる
- **トランジション**: `xfade`フィルターでクロスフェード

## 5. 収益化設計（将来対応・構造だけ今から意識）

### 5.1 収益化を意識したUXフロー

```
① 動画を選択
      ↓
② 設定（冒頭・末尾の秒数）
      ↓
③「エフェクトを適用する」ボタン
      ↓
★ここに広告・待機画面を挿入しやすい★
  （処理中の待ち時間 = 自然に見てもらえるタイミング）
      ↓
④ ダウンロード
```

### 5.2 将来の収益化オプション

| 方法 | 難易度 | 導入タイミングの目安 |
|---|---|---|
| 処理中に広告（Google AdSense） | 低 | ユーザーが増えてきたら |
| 無料版（透かし入り）/ 有料版（透かしなし） | 中 | 月間数百人使ったら |
| 月額サブスク（高画質・長尺対応） | 高 | ファンがついてから |

> **Phase 1では収益化機能は実装しない。**
> ただし処理中の待機画面は「広告を後から挿入しやすい構造」で作っておく。

## 7. 画面構成（Phase 1）

### 7.1 メイン画面のUI構成（スマホ縦型）

```
┌──────────────────────┐
│      PastelClip       │
│  ─────────────────  │
│                       │
│  [ 動画を選択する ]    │
│  （タップで選択）      │
│                       │
│  冒頭の実写時間        │
│  [────●────] 2秒    │
│                       │
│  末尾の実写時間        │
│  [────●────] 2秒    │
│                       │
│ [ エフェクトを適用する ]│
│                       │
│  ─────────────────  │
│  ⏳ 処理中...          │
│  [████████░░] 80%    │
│  ★ここに広告スペース★  │
│  ─────────────────  │
│  ✅ 完成！             │
│  [ 動画をダウンロード ] │
└──────────────────────┘
```

### 7.2 UXフロー

```
① 動画を選択（mp4・mov対応）
      ↓
② 冒頭・末尾の実写時間をスライダーで設定
      ↓
③「エフェクトを適用する」をタップ
      ↓
④ プログレスバー表示（ブラウザ内で変換中）
      ↓
⑤「ダウンロード」ボタンが表示される
      ↓
⑥ ダウンロード完了 → TikTok等に投稿
```

## 9. 制約・考慮事項

| 項目 | Phase 1 での対応 |
|---|---|
| 対応形式 | mp4・mov（入力）、mp4（出力） |
| 最大ファイルサイズ | 500MB（ブラウザメモリ依存） |
| 最大動画長 | 60秒（ショート動画想定） |
| 処理時間 | PCは数十秒・スマホは1〜2分かかる可能性あり（Phase 2で検証） |
| スマホ対応 | Phase 1からスマホ縦型UIで設計 |
| プライバシー | 動画はサーバーに送られない（ブラウザ内完結） |
